# 🔒 Безопасность и Секретные Данные

## ✅ Что НЕ должно попасть в репозиторий

### Файлы с данными
- `*.log` - логи могут содержать IP-адреса и токены
- `data/servers.json` - кэш серверов
- `data/whitelist.txt` - белый список
- `data/blacklist.txt` - чёрный список
- `config/*.json` - конфигурационные файлы
- `.vpn_proxy.sh` - временный файл с прокси
- `*.desktop` - файлы автозапуска

### Секретные данные в коде
- GitHub токены (`ghp_*`, `github_pat_*`)
- API ключи (`API_KEY`, `apikey`)
- Пароли (`PASSWORD`, `SECRET`)
- Приватные ключи

## 🛡️ Как безопасно обновлять репозиторий

### 1. Используйте скрипт push-safe.sh

```bash
./push-safe.sh
```

Скрипт автоматически:
- ✅ Проверяет наличие секретов
- ✅ Исключает лишние файлы
- ✅ Делает коммит
- ✅ Отправляет в репозиторий

### 2. Ручная проверка

Перед коммитом проверьте:

```bash
# Поиск токенов
grep -r "ghp_\|github_pat" *.py

# Поиск API ключей
grep -r "API_KEY.*=" *.py

# Проверка staged файлов
git diff --cached --name-only
```

### 3. Если случайно закоммитили секрет

```bash
# Отменить последний коммит (секрет ещё в истории)
git reset --soft HEAD~1

# Или полностью очистить историю (если секрет давно)
git filter-branch --force --index-filter \
  'git rm --cached --ignore-unmatch ПУТЬ_К_ФАЙЛУ' \
  --prune-empty --tag-name-filter cat -- --all

# Отправить изменения
git push origin --force --all
```

## 📋 Чеклист перед публикацией

- [ ] Удалены все GitHub токены
- [ ] Удалены API ключи и пароли
- [ ] Файлы `.log` исключены из git
- [ ] `servers.json` не в репозитории
- [ ] `whitelist.txt` и `blacklist.txt` не в репозитории
- [ ] `.gitignore` настроен правильно
- [ ] Логи не содержат персональные данные

## 🔍 Проверка безопасности

Запустите перед каждым коммитом:

```bash
# Поиск секретов
grep -rE "ghp_[a-zA-Z0-9]{36}|github_pat_[a-zA-Z0-9]{22,}|API_KEY.*[a-zA-Z0-9]{20,}" . --include="*.py" --include="*.sh"

# Проверка .gitignore
git check-ignore -v *.log data/servers.json
```

## 🚀 Безопасное обновление

```bash
# 1. Остановить VPN
pkill -f "vless-vpn"

# 2. Очистить логи
rm -f ~/vpn-client/logs/*.log

# 3. Запустить безопасный пуш
cd ~/vpn-client-aggregator
./push-safe.sh
```

## ⚠️ Важно!

Если вы случайно закоммитили секрет:
1. **Немедленно** удалите токен в GitHub Settings
2. Создайте новый токен
3. Очистите историю git
4. Отправьте изменения

**Никогда не храните секреты в коде!** Используйте переменные окружения или отдельные файлы конфигурации, которые не попадают в git.
