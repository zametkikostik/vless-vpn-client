## 🔧 НАСТРОЙКА БРАУЗЕРА

### Firefox (ПОЛНАЯ ЗАЩИТА)

#### 1. Настройка прокси:
1. Откройте **Настройки**
2. Прокрутите до **Параметры сети**
3. Нажмите **Настроить...**
4. Выберите **Ручная настройка прокси**
5. Заполните:
   - **SOCKS5 прокси:** `127.0.0.1`
   - **Порт:** `10808`
   - ✅ **DNS через SOCKS v5** (ВАЖНО!)
6. Нажмите **OK**

#### 2. Проверка:
1. Откройте https://dnsleaktest.com
2. Запустите **Standard Test**
3. Должны видеть сервера VPN, а не провайдера!

#### 3. Дополнительная защита:
```
about:config → поиск → dns

network.proxy.socks_remote_dns = true
network.trr.mode = 0
```

---

### Chrome/Chromium

#### Запуск с защитой DNS:
```bash
google-chrome \
  --proxy-server="socks5://127.0.0.1:10808" \
  --host-resolver-rules="MAP * ~NOTFOUND" \
  --async-dns
```

#### Или расширение:
1. Установите **Proxy SwitchyOmega**
2. Настройте SOCKS5: `127.0.0.1:10808`
3. Включите **Use SOCKS5 for DNS**

---

### Brave Browser

#### Запуск:
```bash
brave-browser \
  --proxy-server="socks5://127.0.0.1:10808" \
  --host-rules="MAP * ~NOTFOUND"
```

## 🐧 СИСТЕМНАЯ НАСТРОЙКА (ВСЕ ПРИЛОЖЕНИЯ)

### Вариант 1: Проксификация через proxychains

```bash
# Установка
sudo apt install proxychains4

# Настройка
sudo nano /etc/proxychains4.conf
```

**В конце файла добавьте:**
```
socks5 127.0.0.1 10808
```

**Использование:**
```bash
# Запуск приложения через VPN
proxychains4 firefox
proxychains4 google-chrome
proxychains4 telegram-desktop
```

---

### Вариант 2: Системный прокси (не рекомендуется)

```bash
# Временная настройка
export http_proxy="http://127.0.0.1:10809"
export https_proxy="http://127.0.0.1:10809"
export all_proxy="socks5://127.0.0.1:10808"

# Для терминала
alias proxy='export all_proxy="socks5://127.0.0.1:10808"'
alias unproxy='unset all_proxy http_proxy https_proxy'
```

## 📱 НАСТРОЙКА НА ТЕЛЕФОНЕ

### Android через Wi-Fi:

1. **Настройки** → **Wi-Fi**
2. Долгое нажатие на вашу сеть
3. **Изменить сеть** → **Дополнительно**
4. **Прокси-сервер:** Вручную
5. **Имя хоста:** IP вашего компьютера в локальной сети
6. **Порт:** `10809` (HTTP) или используйте приложение

### Android через приложение:

1. Установите **v2rayNG** или **Hiddify**
2. Импортируйте конфиг
3. Подключитесь

## 🔍 ПРОВЕРКА ЗАЩИТЫ

### 1. Проверка DNS (ВАЖНО!):

**Без VPN:**
```bash
nslookup google.com
# Должен показать DNS провайдера
```

**С VPN через SOCKS5h:**
```bash
curl -x "socks5h://127.0.0.1:10808" https://api.ipify.org?format=text
# Должен показать IP сервера VPN
```

### 2. Онлайн проверка:

**Откройте в браузере с настроенным прокси:**

1. **DNS утечка:**
   - https://dnsleaktest.com
   - https://dnsleak.com
   - ✅ Должны быть сервера VPN

2. **IP утечка:**
   - https://whoer.net
   - https://ipleak.net
   - ✅ Должен быть IP сервера VPN

3. **WebRTC утечка:**
   - https://browserleaks.com/webrtc
   - ✅ Не должно быть вашего реального IP

## 🚀 МАКСИМАЛЬНАЯ ЗАЩИТА

### 1. Tor Browser (лучшая анонимность):

```bash
# Установка
sudo apt install torbrowser-launcher

# Запуск
torbrowser-launcher
```

**Плюсы:**
- ✅ Полная анонимность
- ✅ DNS через Tor
- ✅ Защита от fingerprinting

**Минусы:**
- 🐌 Медленнее VPN

## 🆘 ЕСЛИ ЧТО-ТО НЕ РАБОТАЕТ

### DNS всё ещё уходит провайдеру:

1. Проверьте **DNS через SOCKS v5** в браузере
2. Используйте **socks5h://** вместо **socks5://**
3. Отключите **DNS-over-HTTPS** в браузере

### IP всё ещё виден:

1. Проверьте **WebRTC утечку**
2. Отключите WebRTC в браузере
3. Используйте расширение **WebRTC Control**

### VPN не работает:

```bash
# Перезапуск
sudo systemctl restart vless-vpn-ultimate

# Проверка логов
tail -20 ~/vless-vpn-client/logs/client.log

# Авто-фикс
python3 /home/kostik/vless-vpn-client/auto-fix-vpn.py
```
